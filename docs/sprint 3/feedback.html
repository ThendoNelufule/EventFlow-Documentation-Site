Overview

We conducted user testing for Sprint 3 on Sunday, 28 September 2025. The purpose of this session was to evaluate usability of the Staff Portal, gather participant feedback, and identify areas for improvement. Below is the participant, contact details, the issue found, and the feedback provided.

Feedback Participant: Musala Ndouvhada

Contact: 083 667 9065

Scenario: Musala went through the full process of testing the Staff Portal, focusing on forms, navigation, and page responsiveness. While testing the staff form, he discovered a typographical error where the label for the location field was written as "locatieon" instead of "location".

Feedback (verbatim):

"Your platform is nice, but you are loading the CSS many times which causes it to take too much time. You should use partials for faster performance. Also — if you add a little spice (like subtle micro-interactions, smoother transitions, and cleaner spacing) the UI will feel much more polished and enjoyable to use."

Evidence of Issue:

Issue later resolved; the typo was corrected to "location".

Team Action / Improvement:

  • Fixed the typo in the staff portal form (changed "locatieon" to "location").
  • Refactored CSS loading to use partials/includes so stylesheets are not duplicated across templates — this reduces redundant downloads and improves page-load performance.
  • Planned small UI enhancements ("spice") including micro-interactions, subtle transitions, and adjusted spacing to make the interface feel smoother and more professional in the next sprint.

Proof of Improvements:

The improvements can be directly observed on the live platform. In addition, git commit logs serve as technical evidence showing both the typo fix and the CSS refactor (commit hashes can be provided if needed).

Common Feedback from Participants

After analyzing the feedback from all participants, the most significant and recurring issue identified was related to performance. Multiple testers noted that page load times were slower than expected. Investigation revealed that the root cause was the platform loading the CSS multiple times across templates instead of using partials. This caused unnecessary duplicate downloads, affecting overall performance.

Resolution: The CSS loading was refactored to use partials/includes, which eliminated redundancy and improved page-load speed. This improvement can be observed directly on the live platform. Additionally, the commit history on GitHub serves as proof of this technical fix.

The participants appreciated the responsiveness after this fix, and this was the main concern raised by most testers.
