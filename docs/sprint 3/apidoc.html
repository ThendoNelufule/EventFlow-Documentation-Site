Partner API Documentation

Overview

Our APIs allow external groups to access Event Flow data for events, staff, and guests.

Authentication is handled via API Key, which must be included in all requests to access these endpoints.

We collaborated with three partner groups to test integrations. While their APIs were not functional during testing, all three groups successfully integrated with our API.

Authentication

All endpoints require a valid API Key to be passed in the headers:

Authorization: Bearer <YOUR_API_KEY>

Requests without a valid API key will return a 401 Unauthorized response.

Partner API Endpoints

GET /api/events

Retrieve all events currently in the system.

curl -H "Authorization: Bearer <YOUR_API_KEY>" https://yourdomain.com/api/events

Response:

[
  {
    "id": "eventId123",
    "name": "Tech Conference 2025",
    "description": "Annual tech event",
    "dateTime": "2025-06-20T09:00:00Z",
    "location": {
      "name": "Convention Center",
      "city": "CityName"
    }
  },
  ...
]

GET /api/staff

Retrieve all staff users across events.

curl -H "Authorization: Bearer <YOUR_API_KEY>" https://yourdomain.com/api/staff

Response:

[
  {
    "id": "staffId123",
    "fullName": "Jane Doe",
    "role": "staff",
    "eventId": "eventId123"
  },
  ...
]

GET /api/guests

Retrieve all registered guests.

curl -H "Authorization: Bearer <YOUR_API_KEY>" https://yourdomain.com/api/guests

Response:

[
  {
    "id": "guestId123",
    "fullName": "John Smith",
    "email": "john@example.com",
    "eventId": "eventId123",
    "checkedIn": false
  },
  ...
]

Notes

  • Endpoints are read-only for external partners; no write operations are currently exposed.
  • Ensure your API Key is kept confidential to prevent unauthorized access.
  • During testing with three groups namely CODEXA, PIXEL PERFECT, and MIGHTY MORPHIN, their APIs were unavailable, but all successfully consumed data from our API.
