EvenFlow User Stories - Sprint 3

These user stories describe what each type of user can do in the system, including features up to Sprint 3.

Manager User Story

Manager should be able to:

  • Sign up and log in
  • Log out
  • Set up an event
  • Add staff members for an event
  • Chat in real-time with staff members
  • See the list of venues available
  • Choose a venue
  • Add a venue of their own choice
  • Upload a map linked to a venue
  • Annotate the map for staff and guests
  • Invite guests
  • Add program of the event
  • Export the program as PDF
  • Assign tasks to staff members
  • Make announcements in real time to all staff members

Staff User Story

Staff should be able to:

  • Sign up and log in
  • Log out
  • Chat in real-time with manager and other staff members
  • Receive tasks from the manager
  • Mark tasks as completed
  • Report incidents
  • View incidents reported by other staff and guests
  • Respond to incidents when needed
  • Receive real-time notifications from the manager
  • View the event venue map and see manager annotations

Guest User Story

Guest should be able to:

  • Receive the invitation email with a QR code
  • Present the QR code to the security for check-in
  • Access the event details (schedule, venue, description)
  • View the event venue map and see manager annotations
  • Report incidents during the event

Security User Story

Security should be able to:

  • Scan the QR code of the guests
  • Get notified if the guest is valid or not after scanning
