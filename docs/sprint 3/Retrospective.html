Sprint Retrospective & Improvements

What went well

  • Each member completed their assigned responsibilities on time.
  • First successful exposing APIs to other systems.
  • Map annotation feature was completed and fully functional across roles.
  • Collaboration improved compared to Sprint 2; conflicts were resolved quickly.

What we could improve next sprint

  • Improve testing and debugging time to reduce last-minute bug fixing.
  • Hold more frequent updates to avoid backend integration conflicts.
  • Improve documentation while coding to make API usage clearer.

Why we could not complete everything

  • Minor bugs in UI and backend alignment took extra time to fix.
  • Delay in understanding external API integration slowed down progress.

Next Sprint Plan

  • Polish staff UI and fix responsiveness issues.
  • Enhance error handling and security for API integrations.
  • Conduct user testing sessions for feedback before Sprint 4 review.
