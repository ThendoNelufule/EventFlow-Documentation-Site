Project Management - Sprint 3

Methodology Used

We continued to strictly follow Agile Scrum for Sprint 3. The team focused on planning, execution, and review cycles while ensuring transparency, accountability, and proper task management.

Sprint Planning

Sprint 3 planning involved setting clear goals and assigning tasks to individual team members using Notion. Each task had defined deliverables and acceptance criteria:

  • Backlog items were reviewed and refined before the sprint
  • Tasks were assigned to specific team members with priorities and estimated effort
  • Dependencies and blockers were identified early
  • GitHub was used to manage repository branches, issues, and pull requests for each task

Daily Stand-ups

We held daily meetings via video calls to provide updates and maintain team alignment. Each member shared:

  • Tasks completed the previous day
  • Tasks planned for the current day
  • Any blockers or issues needing team attention

This practice helped reduce confusion and kept the sprint on track.

Task Tracking

All tasks were tracked in Notion using a workflow board with the following stages:

  • To Do
  • In Progress
  • Review / Testing
  • Done

GitHub was used to track code changes, manage pull requests, and review commits. This ensured accountability and traceability of all changes.

Evidence & References

  • Sprint planning, task tracking, and daily stand-ups were documented in our Notion workspace
  • All code-related changes were tracked via GitHub repository commit logs

Rationale

Strictly following Agile Scrum in Sprint 3 ensured that:

  • We delivered incremental working features consistently
  • User feedback could be incorporated quickly into development cycles
  • Transparency and task accountability were maintained using Notion and GitHub
  • The methodology was followed strictly, providing clear evidence of process adherence
