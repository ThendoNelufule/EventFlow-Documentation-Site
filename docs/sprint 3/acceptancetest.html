EvenFlow Acceptance Tests - Sprint 3

The following acceptance tests verify that each new user story introduced in Sprint 3 works as intended.

Manager Acceptance Tests

  • Task Assignment: Given a manager is logged in, when they assign a task to a staff member, then the task should appear in that staff memberâ€™s task list.
  • Map Upload: Given a manager has selected a venue, when they upload a map, then the map should be stored and linked to the venue.
  • Map Annotation: Given a map is uploaded, when the manager adds annotations, then staff should see the updated annotated map.
  • Announcements: Given staff are online, when the manager posts an announcement, then all staff should instantly receive the announcement in real time.

Staff Acceptance Tests

  • Task Completion: Given a staff member has assigned tasks, when they complete a task and mark it done, then the system should update the task status and notify the manager.
  • Incident Reporting: Given staff are logged in, when a staff member reports an incident, then the incident should be visible to all staff and the manager. Other staff should also be able to mark attendance or follow-up on the incident.
  • Notifications: Given a manager sends a notification, when staff are logged in, then the notification should appear instantly in the staff interface.
  • Map Viewing: Given the manager uploads and annotates a map, when a staff member opens the map, then they should see the venue map along with all manager annotations.

Guest Acceptance Tests

  • Map Access: Given the manager uploads and annotates a map, when a guest opens the event map, then they should be able to view the venue map along with all manager annotations.
  • Event Details: Given a guest is invited to an event, when they access the event page, then they should be able to see the event details such as schedule, location, and description.
  • Incident Reporting: Given a guest attends an event, when they report an incident, then the system should record it and make it visible to staff and the manager for follow-up.
