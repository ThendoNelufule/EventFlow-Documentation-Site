Frontend Testing Documentation - Sprint 2

Overview

For Sprint 2, we implemented Playwright automated tests focused on the frontend of the Event Flow system. The goal was to verify that the user interface works correctly and consistently without flaky behavior.

Test Coverage

The following areas were tested:

  • Navigation Tests: Verified page routing and correct loading of dashboard, chat, and task assignment pages.
  • Chatbox Tests: Confirmed that messages can be typed, sent, and displayed correctly in the chat interface.
  • Task Assignment: Checked functionality of "Assign New Task", "Task Management", and "Team Workload" buttons and forms.
  • Venue Upload: Ensured that image uploads are handled (with background.jpg placed in the visuals folder).

Session & Cookies

Since our tests require an authenticated session, we saved cookies from a logged-in manager account into a file called tests/cookies.json. Each test loads these cookies before navigating to the target page:


const cookies = JSON.parse(fs.readFileSync('tests/cookies.json', 'utf-8'));
await page.context().addCookies(cookies);

This ensures that tests always start with a valid session and do not fail due to expired logins.

Running the Tests

To run all frontend tests and generate an HTML report, use the following command:

npx playwright test tests/ --reporter=html

After running, you can open the report with:

npx playwright show-report

Reliability & Robustness

The test cases were designed to fail only if there is a genuine frontend or backend issue. For example, the venue upload test correctly fails when the backend cannot process image uploads. This demonstrates the robustness of our tests rather than a flaw in their implementation.
