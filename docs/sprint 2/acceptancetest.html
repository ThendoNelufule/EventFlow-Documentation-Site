EvenFlow Acceptance Tests

The following acceptance tests verify that each user story works as intended.

Manager Acceptance Tests

  • Login: Given a manager is registered, when they log in with correct credentials, then they should access the dashboard.
  • Logout: Given a manager is logged in, when they log out, then they should return to the login page.
  • Event Setup: Given a manager is logged in, when they create an event with valid details, then the event should be saved and visible.
  • Staff Management: Given an event exists, when the manager adds staff, then staff should appear in the event staff list.
  • Chat: Given a manager and staff are online, when the manager sends a message, then the staff should see it instantly.
  • Venues: Given the manager opens venue list, when they select or add a venue, then it should be linked to the event.
  • Guests: Given the manager invites guests, when emails are sent, then each guest should receive an invitation with a QR code.
  • Program Export: Given a program exists, when the manager exports it, then a downloadable PDF should be generated.

Staff Acceptance Tests

  • Login: Given staff is registered, when they log in, then they should see assigned events.
  • Logout: Given staff is logged in, when they log out, then they should be redirected to login.
  • Chat: Given staff is online, when they receive a message, then it should appear instantly in their chat window.

Guest Acceptance Tests

  • Invitation: Given a guest is invited, when the manager sends the invite, then the guest should receive an email with a unique QR code.
  • Check-in: Given a guest arrives at the event, when they present their QR code, then it should validate successfully.

Security Acceptance Tests

  • Valid QR Code: Given a guest presents a valid code, when security scans it, then the system should display success.
  • Invalid QR Code: Given a guest presents an invalid code, when security scans it, then the system should display a warning.
