1. Purpose

Build a website that supports real-time event management on the day of the event — including guest check-ins, schedule monitoring, and live updates. The system is modular: UI modules for operators on-site, APIs for integration, and a data layer to persist operational state.

2. Core Features

Guest Check-In

  • Check guests in via manual search or built-in QR generation.
  • Upload guest lists manually if integration is unavailable.

Schedule & Announcements

  • Push the program of the event to staff and guests
  • View and edit a real-time schedule for events or sessions.
  • Push announcements to displays or message boards.

Event Venues

  • Display venues or upload venues of interest.
  • Venues come with their own floor maps
  • Use drawing tools to annotate in real-time.

On-Site Coordination

  • Assign tasks to staff, log incidents, and mark task completion.
  • Offline support for mobile teams.

communication

  • Manager and their staff can chat real-time

3. UI Modules

Check-In Console

Scan or search guests; manually mark arrivals.

  • QR scan or name/email search
  • Manual override for edge cases
  • Immediate status feedback

Live Schedule View

Editable agenda; broadcast announcements.

  • Drag-to-reorder sessions
  • Live status (On time / Delayed / Cancelled)
  • Push announcements to boards

Venue Map

Upload static floorplans; add labels or emergency notes.

  • POI tags (stages, exits, booths)
  • Real-time drawings/annotations
  • Versioned map uploads

Staff Dashboard

Assign roles, log incidents, and chat in real-time.

  • Task assignment & completion
  • Incident logging
  • Team chat (per event)

4. API Modules

Check-In API

Update and view guest entry data.

PATCH /api/checkin/{guestId} GET   /api/checkin?eventId={id}

Schedule API

Modify and retrieve live session data.

GET   /api/schedule?eventId={id} PUT   /api/schedule/{sessionId}

Map API

Serve event layout and allow annotations.

POST /api/maps/upload POST /api/maps/{mapId}/annotations

Staff API

Handle task assignments, completion, and internal logs.

POST /api/staff/tasks PATCH /api/staff/tasks/{taskId}

Communication API

Send messages and notification between manager, staff and guests

POST /api/manager/chat POST /api/staff/team-chat

6. System Flow (Day-of Event)

  1. Pre-event: Import guest list, set schedule, upload maps, create staff tasks.
  2. Doors open: Guests checked in via QR/manual; status updates in real time.
  3. Live operations: Schedule changes propagate; announcements broadcast to boards.
  4. On-site coordination: Tasks assigned/completed; incidents logged; team chat active.
  5. Wrap-up: Export check-in stats, final schedule, incident summary.
